data/browsers.json
~~~~~~~~~~~~~~~~~~

This file contains data on Web browser environment.

* Structure

The file contains a JSON object, which has following name/value pairs:

  referrer_policies [object]

    Names are referrer policy values and values are corresponding
    [protocol-object] descriptions, with following additional
    name/value pairs:

      attr [string?]

        Whether the value can be specified in a referrer policy
        attribute (i.e. HTML |referrerpolicy| attribute).  If |MAY|,
        it is allowed.  Otherwise, it is not allowed.

      attr_keyword_url [string?]

        The URL of the specification where the value is defined as an
        attribute value of the referrer policy attribute, if any.

      attr_state [string?]

        The name of the enumerated attribute state corresponding to
        the referrer policy value, if any, without " state" suffix.

      http [string?]

        Whether the value can be specified in an HTTP
        |Referrer-Policy:| header.  If |MAY|, it is allowed.
        Otherwise, it is not allowed.

      idl [string?]

        Whether the value can be specified where Web IDL
        |ReferrerPolicy| enumeration value is expected.  If |MAY|, it
        is allowed.  Otherwise, it is not allowed.

      meta [string?]

        Whether the value can be specified in HTML |<meta
        name=referrer>|'s |content| attribute.  If |MAY|, it is
        allowed.  Otherwise, if |MUST NOT|, it is not allowed but a
        browser must support it.  Otherwise, it is not allowed.

      preferred [string?]

        The preferred referrer policy value, if any.

      rel [string?]

        The link type value, in lowercase cannonical form,
        corresponding to the referrer policy, if any.

  task_sources [object]

    List of known task sources.  Names are names of task sources and
    values are JSON objects containing descriptions of them, using
    following name/value pairs:

      id [id-in-spec?]

        The ID for the task source in the specification of |spec|, if
        known.

      multiple [boolean]

        Whether there can be multiple task sources of this kind for an
        event loop, corresponding to relevant objects.

      spec [spec?]

        The identifier of the specification in which the task source
        is defined, if known.

      status [status?]

        The status of the task source, if known.
